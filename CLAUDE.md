# CLAUDE.md

## ドキュメント管理

コードを修正・追加した場合は、`doc/` 配下の仕様ドキュメントも必ず同時に更新すること。

- 新しい画面を追加した場合 → `doc/04_screens.md` を更新
- API エンドポイントを追加・変更した場合 → `doc/03_api.md` を更新
- データモデル・スキーマを変更した場合 → `doc/02_data_model.md` を更新
- 認証・権限フローを変更した場合 → `doc/05_auth_flow.md` を更新
- プロジェクト構成・技術スタックを変更した場合 → `doc/01_overview.md` を更新
- 複数ドキュメントにまたがる変更の場合は、影響する全ファイルを更新すること

## vte.cx 仕様の参照先

vte.cx BaaS の仕様（SDK の使い方・エンティティ・ACL・認証など）を確認する場合は、  
プロジェクト直下の `README.md` を参照すること。
