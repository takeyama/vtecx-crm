## クイックリファレンス

### ロール別操作権限

| 操作 | 管理者 | 営業担当者 | 閲覧者 |
|---|:---:|:---:|:---:|
| 全データ閲覧 | ✓ | ✓ | ✓ |
| CSV エクスポート（顧客・商談） | ✓ | ✓ | ✓ |
| 顧客 登録/編集/削除 | ✓ | ✓ | — |
| 顧客 CSV インポート | ✓ | ✓ | — |
| 商談 登録/編集/削除 | ✓ | ✓ | — |
| 担当者・対応履歴 登録/編集/削除 | ✓ | ✓ | — |
| ユーザー管理・インデックス再適用 | ✓ | — | — |
| プロフィール編集 | ✓ | ✓ | ✓ |

> フロントエンドは `canWrite = isAdmin || isSales` でボタン表示を制御（デフォルト非表示）。  
> サーバー側は vte.cx ACL（`contributor`・`folderacls`）が最終的なアクセス制御を担う。

### 主要 URL

```
/              ダッシュボード
/customer      顧客一覧
/deal          商談一覧
/settings      ユーザー設定
/admin/users   ユーザー管理（管理者専用）
/setup         初回表示名設定（未設定ユーザーのみ）
/pending       権限付与待ち（未割り当てユーザーのみ）
```

## これから実装すべき機能

### 優先度について

| 記号 | 意味 |
|---|---|
| 🔴 高 | UX上の欠損・実運用上の必須機能 |
| 🟡 中 | あると便利・競合CRMとの差別化 |
| 🟢 低 | 長期的な強化・高コスト機能 |

---

### 🔴 高優先度（すべて✅実装済み）

#### 1. 既存データへのインデックス適用 ✅

**背景**: vte.cx のインデックスは設定後に登録したデータにのみ適用される。設定前の既存顧客・商談データは検索結果に表示されない。

**対応方針**:
- 管理者向けの「インデックス再構築」API を実装する
- 全エントリを取得して PUT し直すバッチ処理を追加する

```typescript
// /api/admin/reindex のイメージ
const entries = await vtecxnext.getFeed('/crm/customer')
for (const entry of entries) {
  await vtecxnext.put({ feed: { entry: [entry] } })
}
```

**影響ファイル**: `src/app/api/(vtecx)/admin/` 以下に新規追加

---

#### 2. ダッシュボード強化 ✅

**背景**: 現状は静的なサマリーカードのみ。担当者・フェーズ別の集計がなく、営業進捗の把握が難しい。

**実装内容**:
- 商談フェーズ別の件数・合計金額バーチャート
- 担当者別の進行中商談数
- クローズ予定日が今週・来週の商談アラートリスト
- 次アクション日が本日の対応履歴リスト（`next_action_date` インデックス活用）

**技術メモ**:
- グラフは `recharts` または MUI の BarChart を使用
- `next_action_date` の当日フィルタ: `/crm/customer?f&activity.next_action_date-eq-{today}`（要インデックス）

---

#### 3. 顧客・商談のフリーワード検索 ✅

**背景**: 現状は完全一致フィルタのみ。顧客名・商談名の部分一致検索がない。

**実装内容**:
- 顧客一覧に検索テキストボックスを追加
- `name-ft-{keyword}` の全文検索クエリを使用

**前提条件**:
- `template.xml` の `<rights>` に全文検索インデックス（`;`）を追加する必要がある
  ```
  customer.name;/crm/customer
  deal.name;/crm/deal
  ```
- `pnpm upload:template` でサーバーに反映後に動作する

---

#### 4. 商談一覧のクローズ日フィルタ ✅

**背景**: 商談一覧のフィルタにクローズ予定日の範囲指定がない。期限管理ができない。

**実装内容**:
- 商談一覧に「クローズ予定日（from〜to）」の日付レンジ入力を追加
- `deal.expected_close_date-ge-{from}&deal.expected_close_date-le-{to}` クエリを使用（インデックス設定済み）

---

### 🟡 中優先度

#### 5. タスク管理

**背景**: 担当者への TODO・リマインダーを管理する仕組みがない。

**新規エンティティ**:
```
task
 title(string){255}!
 due_date(date)
 customer_uri(string){500}
 deal_uri(string){500}
 assigned_uid(string){255}
 status(string){20}!    ← todo / done
 is_deleted(boolean)
```

**新規パス**:
- `/crm/task/{tid}` — タスク一覧
- `folderacls.json` に `/crm/task` を追加

**新規画面**:
- `/task` — タスク一覧（自分のタスク、期限でソート）
- ダッシュボードに「今日のタスク」ウィジェット追加

---

#### 6. CSV エクスポート（インポートは✅実装済み）

**実装済み**:
- 顧客一覧「CSVインポート」ボタン（`CsvImport.tsx`）
  - UTF-8（BOM付き）/ Shift-JIS 自動判定
  - インポート前プレビュー・進捗バー・結果表示
  - テンプレート CSV ダウンロード
  - 権限制御済み（canWrite のみ表示）

**未実装（エクスポート）**:
- 顧客一覧・商談一覧の「CSV ダウンロード」ボタン
- フィルタ適用後のデータをエクスポート
- `Blob` + `URL.createObjectURL` でブラウザダウンロード

---

#### 7. 商談パイプライン（カンバンビュー）

**背景**: 商談の進捗をフェーズ別に俯瞰できる画面がない。

**実装内容**:
- 商談一覧に「リスト表示 / カンバン表示」切り替えボタンを追加
- フェーズをカラムとした Kanban ボード
- カード内: 商談名・顧客名・金額・クローズ予定日
- ドラッグ＆ドロップによるフェーズ変更（`react-dnd` または `@dnd-kit/core`）

---

#### 8. 活動サマリー定期メール

**背景**: 週次で営業活動のサマリーをメール通知したい。

**実装内容**:
- `setup/_settings/properties.xml` の SendGrid 設定を使用
- 管理者向けに「週次レポート送信」ボタンを追加（手動トリガー）または cron 実装
- 内容: 今週の対応履歴件数・進行中商談・期限超過タスク

---

### 🟢 低優先度（長期）

#### 9. メール・カレンダー連携

**背景**: Gmail や Google Calendar と連携して、メールを対応履歴として自動取り込みしたい。

**実装方針**:
- Claude Code の MCP（Google Calendar・Gmail）を活用
- Gmail MCP: 特定ラベルのメールを対応履歴として登録
- Google Calendar MCP: 商談の予定を Calendar に追加

**参考**: Claude Code の MCP 設定 → `~/.claude/settings.json`

---

#### 10. 名刺管理（OCR）

**背景**: 名刺情報を手入力せず、スキャンして顧客担当者に登録したい。

**実装方針**:
- 名刺画像をアップロード → Claude API（Vision）で OCR 解析
- 解析結果を顧客担当者の登録フォームに自動補完

**技術メモ**:
- Claude API の `claude-opus-4-7` モデルを使用（画像入力対応）
- vte.cx のファイルストレージへの画像保存方法は要調査

---

#### 11. AI アシスタント

**背景**: 対応履歴・商談情報から次のアクションを AI が提案してほしい。

**実装方針**:
- 顧客詳細・商談詳細に「AI サマリー生成」ボタンを追加
- 対応履歴・商談メモを Claude API に渡し、要約・次アクション提案を生成
- `claude-sonnet-4-6` モデルを使用（コスト・速度バランス）

---

### 既知の課題（バグ・技術的負債）

| 課題 | 詳細 | 対応方針 |
|---|---|---|
| ~~顧客一覧・商談一覧のページング~~ | ✅ 実装済み（前へ/次へボタン + ページ表示） | — |
| インデックス未適用の既存データ | 検索フィルタで既存データが表示されない場合がある | ユーザー管理画面の「インデックス再適用」ボタンで解消 |
| 対応履歴の `deal_uri` 選択 | 現状テキスト入力のまま。商談 Select に変更すべき | DealForm と同様の Select 実装 |
| 商談一覧のCSVエクスポートで顧客名が出力されない | `customer_uri`（パス）のまま出力。顧客名に変換できていない | 顧客一覧を別途取得してマッピング |
