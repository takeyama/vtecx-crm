## ページコンポーネント構造

メイン画面はすべて**外側/内側の2層構造**を採用している。  
詳細は [05_auth_flow.md](./05_auth_flow.md) の「ページコンポーネントの構造パターン」を参照。

```
外側コンポーネント（例: CustomerListPage）
  └── <MainLayout>        ← 認証ガード。state !== 'ok' の間は children を描画しない
        └── <CustomerListContent />  ← データ取得 useEffect はここに書く
```

> **新しいメイン画面を追加する際は必ずこの構造に従うこと。**  
> 外側コンポーネントで `useEffect` によるデータ取得を行うと、  
> `useAuthGuard` より先にデータ API が呼ばれて 403 → ログイン遷移が発生する。

## ナビゲーション構成

共通サイドバー（`MainLayout`）がすべてのメイン画面に表示される。

管理者の場合:
```
┌─────────────────────┐
│ CRM                 │
│ 表示名 or email     │
├─────────────────────┤
│ 📊 ダッシュボード   │  → /
│ 👥 顧客一覧        │  → /customer
│ 🤝 商談一覧        │  → /deal
│                     │
├─────────────────────┤
│ 🔐 ユーザー管理    │  → /admin/users  ← 管理者のみ表示
│ ⚙  設定            │  → /settings
│ 🚪 ログアウト      │  → /login（セッション破棄）
└─────────────────────┘
```

管理者以外の場合:
```
┌─────────────────────┐
│ CRM                 │
│ 表示名 or email     │
├─────────────────────┤
│ 📊 ダッシュボード   │  → /
│ 👥 顧客一覧        │  → /customer
│ 🤝 商談一覧        │  → /deal
│                     │
├─────────────────────┤
│ ⚙  設定            │  → /settings
│ 🚪 ログアウト      │  → /login（セッション破棄）
└─────────────────────┘
```

- デスクトップ（sm以上）: 幅220pxの固定サイドバー
- モバイル（xs）: ハンバーガーメニュー → 一時的ドロワー
- 現在のページのメニュー項目がハイライト
- 「ユーザー管理」は `info.isAdmin === true` の場合のみ表示
- サイドバー上部にログインユーザーの**権限Chip**を表示（管理者=赤・営業担当者=青・閲覧者=グレー・未割り当て=黄）

## 権限制御

### 基本方針

**デフォルト非表示**。権限がある場合のみボタン・操作を表示する。

```typescript
const { info } = useAuthContext()
const canWrite = info?.isAdmin || info?.isSales  // 書き込み権限
```

### 権限マトリクス

| 操作 | 管理者 | 営業担当者 | 閲覧者 |
|---|:---:|:---:|:---:|
| 全データ閲覧 | ✓ | ✓ | ✓ |
| 顧客 登録/編集/削除 | ✓ | ✓ | — |
| 顧客 CSV インポート | ✓ | ✓ | — |
| 商談 登録/編集/削除 | ✓ | ✓ | — |
| 担当者（contact）登録/編集/削除 | ✓ | ✓ | — |
| 対応履歴 登録/編集/削除 | ✓ | ✓ | — |
| 担当者（assigned_uid）変更 | ✓ | ✓ | — |
| ユーザー管理 | ✓ | — | — |

> vte.cx の ACL（`contributor`）がサーバー側でアクセス制御を担う。  
> フロントエンドの権限制御は UX のための表示切り替えであり、  
> 権限のないユーザーが直接 API を呼んだ場合でもサーバー側で 403 を返す。

## 各画面の詳細

### ダッシュボード（`/`）

**目的**: システム全体のサマリーを一覧表示。

**表示内容**:
- サマリーカード3枚（取引中顧客数・進行中商談数・進行中商談合計金額）
- フェーズ別集計パネル（各フェーズの件数・金額 + プログレスバー）
- 今週クローズ予定パネル（7日以内にクローズ予定の商談一覧）
- クイックアクション（権限によってボタン表示切り替え）
- 進行中商談テーブル（クローズ予定日順 5件）

---

### 顧客一覧（`/customer`）

**目的**: 全顧客のリスト表示・登録画面へのナビゲーション。

**表示内容**:
- フィルタバー（顧客名フリーワード・ステータス Select・担当営業 Select + 絞り込みボタン）
  - フリーワード指定時はステータスフィルタは無効（全文検索インデックス使用）
  - **担当営業フィルタ選択時**: エイリアス横断検索（`GET /api/crm/member/{uid}`）を実行。ステータス・フリーワードとの組み合わせ可能。ページングも動作する
  - Enter キーでも検索実行
- 顧客テーブル（会社名・業種・電話番号・ステータス）
- ステータスは色付きChipで表示
- 行クリックで顧客詳細へ遷移
- ページネーション（前へ / X/Nページ / 次へ）: 25件/ページ、最大50ページ分のカーソルを作成

**権限制御** (`canWrite = isAdmin || isSales`):
- 「CSVエクスポート」ボタン — **全ロール**表示（データが0件のとき disabled）
- 「CSVインポート」ボタン — canWrite のみ表示
- 「新規登録」ボタン — canWrite のみ表示

**CSVエクスポート** (`src/utils/csvutil.ts`):
- 現在表示中のデータ（フィルタ適用済み）をそのまま CSV ダウンロード
- BOM 付き UTF-8 で出力（Excel でそのまま開ける）
- 出力列: 顧客名・カナ・業種・企業規模・電話・FAX・郵便番号・住所・Web・ステータス・獲得経路・年間売上・メモ
- ファイル名: `customers_YYYY-MM-DD.csv`

**CSVインポート** (`CsvImport.tsx`):
- UTF-8（BOM付き）/ Shift-JIS 自動判定
- インポート前のプレビュー表示（先頭5件）
- **全件を1トランザクションでバルク登録**（`POST /api/crm/customer/bulk`）
- テンプレート CSV ダウンロード機能
- 列マッピング: `顧客名`→`name`、`ステータス`→`status`（日本語値を英語キーに変換）など

---

### 顧客詳細（`/customer/[cid]`）

**目的**: 顧客情報・担当者・対応履歴・関連商談をまとめて管理。

**表示内容**:
- 顧客基本情報カード
  - 担当営業欄（複数人対応）: Chip 一覧で表示。canWrite のとき追加ボタン・削除（×）ボタンあり。ユーザー一覧 Select から選択して追加
- タブ切り替え: 「担当者」「対応履歴」「商談」
- 担当者タブ: 一覧表示 + 登録/編集/削除モーダル
  - 登録: ユーザー一覧 Select → 選択時に姓・名・カナ・部署・役職・メール・電話・携帯を自動補完
  - 編集: 各フィールドを直接入力
- 対応履歴タブ: 一覧表示 + 登録/編集/削除モーダル（`next_action_date` 付き）
- 商談タブ: `customer_uri` でフィルタした商談一覧（商談名・フェーズ・金額・予定クローズ日）

**権限制御** (`canWrite = isAdmin || isSales`):
- 「商談登録」ボタン（ヘッダー） — canWrite のみ表示
- 「編集」「削除」ボタン — canWrite のみ表示
- 担当営業欄: 「追加」ボタン・削除（×）アイコン — canWrite のみ表示
- 担当者タブ: 「担当者追加」ボタン・編集/削除アイコン — canWrite のみ表示
- 対応履歴タブ: 「対応履歴追加」ボタン・編集/削除アイコン — canWrite のみ表示
- 商談タブ: 「商談登録」ボタン — canWrite のみ表示

---

### 顧客登録・編集（`/customer/new`, `/customer/[cid]/edit`）

**目的**: 顧客情報のフォーム入力。

**共通コンポーネント**: `CustomerForm.tsx`

**フィールド**: 会社名（必須）・カナ・業種・企業規模・電話・FAX・郵便番号・住所・Web・ステータス（必須）・獲得経路・年間売上・メモ

> 担当者（`assigned_uid`）は顧客詳細画面のインライン編集からのみ変更可。登録・編集フォームには含まない。

---

### 商談一覧（`/deal`）

**目的**: 全商談のリスト表示。

**表示内容**:
- フィルタバー（商談名フリーワード・フェーズ Select・クローズ日From〜To + 絞り込みボタン）
  - フリーワード指定時は他フィルタは無効（全文検索インデックス使用）
  - クローズ日フィルタは `deal.expected_close_date` インデックスを使用
- 商談テーブル（商談名・フェーズ・金額・確度・予定クローズ日）
- フェーズは色付きChipで表示
- 行クリックで商談詳細へ遷移
- ページネーション（前へ / X/Nページ / 次へ）: 50件/ページ

**権限制御**:
- 「CSVエクスポート」ボタン — **全ロール**表示（データが0件のとき disabled）
- 「新規登録」ボタン — canWrite のみ表示

**CSVエクスポート**:
- 出力列: 商談名・顧客URI・フェーズ（日本語）・金額・確度・予定/実際クローズ日・担当者UID・メモ
- ファイル名: `deals_YYYY-MM-DD.csv`

---

### 商談詳細（`/deal/[did]`）

**目的**: 商談情報の表示。

**表示内容**:
- 商談基本情報（フェーズ・金額・確度・予定クローズ日・担当者UID・顧客リンク・メモ）
- 関連顧客へのリンク（クリックで顧客詳細へ遷移）

**権限制御**:
- 「編集」「削除」ボタン — canWrite のみ表示

---

### 商談登録・編集（`/deal/new`, `/deal/[did]/edit`）

**共通コンポーネント**: `DealForm.tsx`

**フィールド**: 商談名（必須）・顧客（必須・Select）・担当者（Select）・フェーズ（必須）・金額・確度・予定クローズ日・メモ

**顧客選択**:
- 顧客は Select ドロップダウンから選択（`fetchAllCustomers` で全件取得）
- 顧客詳細から遷移した場合（`?customer={cid}`）: 顧客が選択済みかつ変更不可（disabled）、キャンセルで元の顧客詳細に戻る

**担当者選択**:
- 顧客選択後に有効化（顧客未選択・ロード中は disabled）
- 顧客に紐付く担当者一覧から Select。顧客変更時に自動リセット

---

### ユーザー設定（`/settings`）

**目的**: 自分のアカウント情報確認とプロフィール編集。

**表示内容**:
- アカウント情報（UID・メールアドレス・現在の権限Chip）
- プロフィールフォーム + 保存ボタン
  - 表示名（必須）・姓・名・姓カナ・名カナ・部署・役職・電話番号・携帯番号

> プロフィールフィールドは `contact`（顧客担当者）エンティティと共通。担当者登録時にここで入力した情報が自動補完される。  
> 保存時にログインメールアドレス（`email`）も userprofile に自動書き込みされ、担当者選択時の補完に使われる。

---

### ユーザー管理（`/admin/users`）

**目的**: 管理者がユーザーに営業担当者・閲覧者権限を付与・剥奪する。

**表示内容**:
- ユーザーテーブル（UID・表示名・営業担当者列・閲覧者列）
- 権限あり: カラーChip（`×` ボタンで削除）
- 権限なし: `+` アイコンボタン（クリックで付与）
- 操作は即時反映
- 「インデックス再適用」ボタン: `POST /api/admin/reindex` を呼び出し、インデックス設定前に登録された既存データへインデックスを適用する

---

### 初回表示名設定（`/setup`）

**目的**: 初回ログイン時に表示名を設定させる。

**条件**: `display_name` が未設定のユーザーが対象。

**フロー**:
1. 表示名を入力して「次へ」
2. 保存後、権限の有無に応じてリダイレクト
   - 権限あり → `/`（ダッシュボード）
   - 権限なし → `/pending`（権限付与待ち）

---

### 権限付与待ち（`/pending`）

**目的**: 権限が付与されるまでユーザーを待機させる。

**条件**: `display_name` 設定済みだがロールなしのユーザーが対象。

**機能**:
- 「管理者に権限付与を依頼してください」メッセージ
- カウントダウン（10秒）付き自動ポーリング
- 「今すぐ確認する」ボタン
- 権限が付与されると自動的にダッシュボードへ遷移
